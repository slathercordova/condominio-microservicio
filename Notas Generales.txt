//  Como se creo este proyecto de microservicios
Crear un proyecto padre Condominio

Luego modificar el pom para indicarle que no es proyecto, solo un agrupador con este comando <packaging>pom</packaging>

Luego crear el primer módulo para el servicio person: new->module, configurar su propio pom y archivo dockerfile,
crear un servicio random de hola mundo para verificar que la configuración este correcta y corran los servicios.

Crear el dockercompose en la raíz de todo el proyecto principal que orgestará todo el proyecto de los N servicios.

Ejecutar los siguientes comandos para levantar el docker

🧠 REGLA DE ORO
Cambiaste	                Qué haces
Primera vez                 mvn clean install + docker-compose up --build
Cambios regulares	        mvn clean package -DskipTests + docker-compose up --build
Cambios pequeños            mvn package + docker-compose restart nombre-service
Solo config (yaml/env)	    docker-compose up --build
Nada (solo levantar)	    docker-compose up
Recompilar 1 módulo         mvn clean package -DskipTests + docker-compose up -d --build edificio-service

//  Comandos Git
1. Posicionarse en la rama master del local
git switch master

2. Hacer pull de lo que hay en la nube
git pull origin master

3. Crear una nueva rama local para las feature
git switch -c feature/nombre-feature

4. Realizar cambios y hacer commits frecuentes
git add .
git commit -m "mensaje"

5. Terminado el trabajo volver a master y hacer pull de la rama feature a la master
git switch master

6. Actualizar master antes del merge
git pull origin master

7. Fusionar feature hacia la master
git merge feature/nombre-feature

8. Subir los cambios al repositorio en la nube
git push origin master

9. Borrar la rama feature después del push (opcional)
git branch -d feature/nombre-feature (local)
git push origin --delete feature/nombre-feature (remoto)

COMANDOS ADICIONALES

- Ver estado actual
git status

- Ver historial de logs
git log
git log --oneline
git log --oneline --graph --decorate --all

- Ver diferencias antes del commit
git diff

- Listar ramas locales y remotas
git branch -a

- Subida inicial
git push -u origin master

- Traer cambios sin fusionar
git fetch
